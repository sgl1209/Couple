# 心事墙 - 实时通知功能

## 功能说明

心事墙是一个两个人可以互相分享心事的地方。与情书不同的是：

- **情书墙**：精心撰写的正式表达，带标题和发布时间
- **心事墙**：可以自由地分享日常的想法、烦恼、开心的事等

## 关键特点

✅ **实时推送** - 一方发布心事，另一方立刻看到（无需刷新）
✅ **浏览器通知** - 收到新心事时会弹出桌面通知
✅ **双向同步** - 在两个不同的浏览器标签页中，变化实时同步
✅ **私密性** - 数据存储在本地 SQLite 数据库，只有两个人能看到

## 使用方法

### 基础使用

1. **登录** → 进入首页
2. **导航菜单** → 点击「心事墙」
3. **写下心事** → 在文本框里输入内容
4. **点击发布** → 心事立刻出现在列表中，并推送给配对人

### 浏览器通知

首次访问心事墙时，浏览器会询问是否允许通知。建议选择「允许」，这样：
- 当配对人发布心事时，桌面右下角会弹出通知
- 即使浏览器在后台也能收到提示

## 技术细节

### 数据存储

```
site.db
└── secretdiary 表
    ├── id (主键)
    ├── content (心事内容)
    ├── author (作者名字)
    └── created_at (发布时间)
```

### 实时通信

- 使用 **WebSocket** (Socket.IO) 实现两端通信
- 用户 A 发布心事 → 服务器通过 `socketio.emit('new_diary')` 广播给所有连接的用户
- 用户 B 接收事件 → 前端实时渲染新卡片

### 前端流程

```
用户发布心事 → POST /secretdiary/post
            ↓
         服务器处理
            ↓
  socketio.emit('new_diary') ← 广播给所有连接客户端
            ↓
   所有打开心事墙的用户接收
            ↓
  前端渲染新卡片 + 触发浏览器通知
```

## 生产环境注意事项

### WebSocket 兼容性

生产环境需要确保服务器支持 WebSocket：

```bash
# 推荐方案：gunicorn + eventlet
pip install gunicorn eventlet
gunicorn --worker-class eventlet -w 1 -b 0.0.0.0:5000 app:app
```

### 反向代理配置（Nginx）

如果用 Nginx 反代，需要配置 WebSocket 支持：

```nginx
location / {
    proxy_pass http://127.0.0.1:5000;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_set_header Host $host;
    proxy_buffering off;
}
```

### 安全考虑

- 数据存储在本地数据库，不经过第三方服务
- 连接仅限于登录用户（`session.get('user_id')`）
- XSS 防护：前端对用户输入进行 HTML 转义

## 常见问题

**Q: 如果一个人离线，另一个人发的心事还会保存吗？**
A: 会的。心事保存在数据库中。离线的人重新进入心事墙时，会看到所有历史记录。

**Q: 能删除已发布的心事吗？**
A: 目前不支持删除功能。如需添加，可参考照片墙的删除逻辑。

**Q: 如果 WebSocket 连接断开了怎么办？**
A: Socket.IO 会自动重连。如果长时间无连接，用户刷新页面即可重建连接。

**Q: 在服务器上部署时，多少人可以同时访问？**
A: 由于只有两个用户，并发问题不大。但如果未来扩展用户，需考虑使用 Redis 作为消息队列。
